# DNAI v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a client-side Next.js web app that ingests a MyHeritage raw CSV and produces an interactive health/pharma/traits report with "fou malade" visualizations.

**Architecture:** Next.js 15 static export, Tailwind + shadcn, Web Worker for parsing/annotation, pure-function annotation lib, d3 + Mol* + react-three-fiber for visualizations. Curated seed data committed; full ClinVar build via optional Python script.

**Tech Stack:** pnpm, Next.js 15, TypeScript strict, Tailwind, shadcn/ui, d3-modular, molstar, three + @react-three/fiber + @react-three/drei, framer-motion, zustand, vitest, biome.

**Reference spec:** [docs/superpowers/specs/2026-04-23-dnai-v1-design.md](../specs/2026-04-23-dnai-v1-design.md)

  - `complement("A") → "T"`, all 4 bases
  - `matchAllele(geno{A,G}, "A", "G") → "ref/alt"`

  - `matchAllele(nocall, ..., ...) → "nocall"`
  - `matchAllele(geno{A,T}, "A", "G") → "ambiguous"` (allele T not ref or alt)
